Bảng nội dung

1. Tổng quan các thành phần (Architecture)

Trước khi cài đặt, bạn cần hiểu rõ các thành phần để không bị bối rối khi trình cài đặt hỏi:

  1. Enterprise Manager (EM):
    • Vai trò: Đây là “bộ não” của hệ thống. Nó là một Server application nhận dữ liệu từ các Agents, lưu trữ, xử lý và cảnh báo.
    • Trong mô hình này: Bạn sẽ cài EM lên con server RHEL riêng biệt.
  2. Agent (Introscope Agent):
    • Vai trò: “Điệp viên” cài trên các máy chủ SAP (Managed System) để thu thập dữ liệu (Memory, CPU, Response Time…) và gửi về EM.
    • Lưu ý: Agent không cài trên máy chủ EM này (trừ khi bạn muốn monitor chính con server EM).
  3. Workstation:
    • Vai trò: Một ứng dụng Desktop (Java client) cài trên Laptop của admin để thiết kế Dashboard, xem metric chi tiết, Debug sâu.
    • Giao diện: Dạng cửa sổ ứng dụng (Rich Client).
  4. WebView:
    • Vai trò: Giao diện Web (truy cập qua Browser). Dùng để xem Dashboard hàng ngày, nhẹ nhàng hơn Workstation.
    • Cài đặt: Thường được tích hợp sẵn trong bộ cài EM.
  5. Management Modules (MM):
    • Đặc biệt của SAP: Bản Wily của SAP có sẵn các “template” dashboard dành riêng cho SAP (như SAP NetWeaver, Host…). Khi cài bản của SAP, các module này tự động được nạp.

2. Chuẩn bị (Prerequisites)

2.1. SAP Notes quan trọng cần đọc

  • Note 797147: Introscope Installation for SAP (Note tổng quan “Thánh kinh”).
  • Note 2909673: Introscope 10.7 Release Notes (Hoặc Note 3247270 cho bản 10.8 nếu bạn dùng bản mới nhất).
  • Note 3247270: Introscope 10.8 Release Notes (Khuyên dùng bản 10.8 nếu tải mới).

2.2. Yêu cầu hệ thống (RHEL)

  • OS: RHEL 7.x, 8.x hoặc 9.x (64-bit).
  • RAM: Tối thiểu 8GB (Khuyến nghị 16GB cho Standalone production).
  • Disk: Tối thiểu 50GB (để lưu trữ dữ liệu SmartStor – database metric của Wily).
  • Java: Wily Introscope đi kèm sẵn JVM (Java Virtual Machine) hoặc yêu cầu cài sẵn SAPJVM 8. Khuyên dùng: Sử dụng JVM đi kèm trong bộ cài (Bundled JVM) để tránh lỗi tương thích.

2.3. Tải bộ cài từ SAP Marketplace

  1. Truy cập: https://launchpad.support.sap.com/#/softwarecenter
  2. Đường dẫn: Support Packages & Patches -> S -> SAP SOLUTION MANAGER -> SAP SOLUTION MANAGER 7.2 -> WILY INTROSCOPE -> WILY INTROSCOPE ENTRPR MGR 10.8 (hoặc 10.7).
  3. Tải file: WILY<version>_LINUX.SAR (Ví dụ: WILYEM10800P_3-70007168.SAR).
  4. Tải tool giải nén: SAPCAR (nếu server chưa có).

3. Quy trình cài đặt chi tiết trên RHEL

Bước 1: Tạo User hệ điều hành

Không bao giờ chạy Wily dưới quyền root. Hãy tạo một user riêng, ví dụ wilyadmin hoặc dùng user sidadm nếu muốn chuẩn SAP.

# Đăng nhập quyền root
useradd -m wilyadmin
passwd wilyadmin
# (Đặt mật khẩu cho user)

Bước 2: Giải nén bộ cài

Upload file .SAR và SAPCAR lên thư mục tạm (ví dụ /tmp/install).

bashcd /tmp/install
chmod +x SAPCAR
./SAPCAR -xvf WILYEM10800P_3-70007168.SAR

Sau khi giải nén, bạn sẽ thấy các file như introscope<version>linux.bin hoặc installer...jar.

Bước 3: Chạy cài đặt (Console Mode)

Chuyển sang user wilyadmin và chạy installer.

Lưu ý: RHEL server thường không có giao diện đồ họa (GUI), nên ta dùng mode console.

su - wilyadmin
cd /tmp/install
# Cấp quyền thực thi
chmod +x introscope10.8.0.0linux.bin

# Chạy cài đặt ở chế độ console
./introscope10.8.0.0linux.bin -i console

Bước 4: Trả lời các câu hỏi trong quá trình cài đặt (Quan trọng)

Màn hình sẽ hiện text, bạn nhập số hoặc text để trả lời:

  1. Introduction: Nhấn Enter để tiếp tục.
  2. License Agreement: Đọc (hoặc lướt qua) và chọn Y (Accept).
  3. Choose Install Folder: Nhập đường dẫn cài đặt.
    • Gợi ý: /home/wilyadmin/introscope hoặc /usr/sap/ccms/wilyintroscope (cần quyền ghi).
  4. Management Module Pack: Installer của SAP sẽ hỏi bạn có muốn cài module SAP không.
    • Chọn: Yes (Đây là điểm khác biệt với bản Wily gốc của Broadcom).
  5. Choose Install Set: Chọn “Full” hoặc “Standalone Enterprise Manager”.
    • Giải thích: Đừng chọn “Clustered” trừ khi bạn cài hệ thống cực lớn với nhiều Collector và 1 MOM (Manager of Managers). Với nhu cầu monitor cơ bản, “Standalone” là đủ.
  6. Port Configuration:
    • Server Port: 6001 (Cổng Agent gửi dữ liệu về – Quan trọng).
    • Web Port: 8080 (Cổng vào WebView/Workstation).
    • HTTPS Port: 8443.
    • Giữ mặc định nếu không xung đột port.
  7. External Component: Nó có thể hỏi về SAP Solution Manager Host.
    • Vì bạn cài độc lập: Bỏ trống hoặc nhập localhost. Chúng ta sẽ không tích hợp ngược lại SolMan.
  8. Password: Đặt mật khẩu cho user Admin. (Nhớ kỹ mật khẩu này để đăng nhập WebView).

Chờ quá trình cài đặt hoàn tất.

4. Cấu hình Sau cài đặt (Post-Installation)

4.1. License (Xử lý vấn đề “Độc lập”)

Mặc định, Wily tải từ SAP sẽ tìm kiếm license.

  • Vào thư mục cài đặt: cd /usr/sap/ccms/wilyintroscope/license
  • Kiểm tra file osgi-*.jar.
  • Lưu ý: Nếu dùng license SAP, nó thường là Open License giới hạn cho SAP. Nếu bạn mua license độc lập từ Broadcom, hãy copy file license.xml vào thư mục này hoặc thư mục config.

4.2. Tinh chỉnh Performance (Heap Size)

Sửa file cấu hình để cấp đủ RAM cho EM (Java process).

  • File: <Install_Dir>/config/IntroscopeEnterpriseManager.lax
  • Tìm dòng: lax.nl.java.option.additional
  • Sửa tham số: -Xms2048m -Xmx4096m (Ví dụ cấp min 2GB, max 4GB RAM).

4.3. Khởi động Wily EM

bashcd <Install_Dir>/bin
./EMCtrl.sh start
  • Kiểm tra trạng thái: ./EMCtrl.sh status
  • Xem log: tail -f <Install_Dir>/logs/IntroscopeEnterpriseManager.log
    • Nếu thấy dòng Introscope Enterprise Manager started, là thành công.

5. Kết nối từ Client (Workstation & WebView)

5.1. Truy cập WebView (Monitor hàng ngày)

  • Mở trình duyệt trên máy tính của bạn.
  • URL: http://<IP_RHEL_Server>:8080/webview
  • User: Admin
  • Pass: (Mật khẩu bạn đặt lúc cài).

5.2. Cài đặt Workstation (Dành cho Admin)

Trên máy tính Windows/Mac của bạn:

  1. Truy cập: http://<IP_RHEL_Server>:8080/install
  2. Tải bộ cài Workstation về máy cá nhân.
  3. Cài đặt và mở lên.
  4. Connect to:
    • Host: <IP_RHEL_Server>
    • Port: 6001
    • User: Admin

6. Tóm tắt quy trình vận hành “Độc lập”

  1. Server RHEL: Chạy duy nhất dịch vụ Introscope EM.
  2. SAP System (Target): Cài Diagnostic Agent (DAA) hoặc Introscope Java Agent (như hướng dẫn ở câu trả lời trước), trỏ introscope.agent.enterprisemanager.transport.tcp.host.DEFAULT về IP của Server RHEL này.
  3. Không cần SolMan: Bạn không cần thực hiện bước “Managed System Configuration” trên SolMan. Dữ liệu sẽ bay thẳng từ Agent -> EM RHEL này.

Bằng cách này, bạn có một hệ thống Monitor chuẩn SAP (nhìn thấy các metric SAP J2EE, ABAP via Host Agent) nhưng hoàn toàn tách biệt khỏi sự cồng kềnh của Solution Manager.
